# Impeccable — marr.github.io

Design context for this repo. Update this file when brand or audience goals change.

## Design Context

### Users

- **Primary visitors:** engineers, hiring managers, and open-source contributors evaluating David Marr’s work.
- **Context:** quick scan of credibility (OSS, stack, writing) and optional deep read (résumé, blog posts).
- **Emotional goal:** **confident clarity** — competent and approachable, never flashy or salesy.

### Brand Personality

- **Three words:** precise, builder-led, quietly ambitious.
- **Voice:** direct product language (what it is, why it matters), not marketing fluff.
- **References to emulate:** crisp technical portfolios and docs (clear hierarchy, strong type, restrained color).
- **Anti-patterns:** generic SaaS landing tropes, purple-gradient hero clichés, crowded stat pills, “AI slop” copy.

### Aesthetic Direction

- **Visual tone:** editorial + engineering — calm surfaces, one strong accent, generous spacing, readable line length.
- **Typography:** **Public Sans** as the single family (already configured); tight tracking on large headings, comfortable body line-height.
- **Theme:** support **light and dark** via Nuxt UI color mode; sepia is used for reading contexts where Content already defines it.
- **Color:** **teal primary** with **slate neutral** (see `app.config.ts`) — aligns with OSS/tooling cues without default indigo/purple.
- **Motion:** subtle, purposeful only (e.g. card hover lift, hero glow); respect `prefers-reduced-motion` where practical.

### Design Principles

1. **Semantic Nuxt UI first** — prefer `text-default`, `text-muted`, `bg-default`, `bg-elevated`, `border-default`, `primary` accents; avoid raw Tailwind palette colors except in documented exceptions.
2. **Composition before chrome** — whitespace and type hierarchy carry the layout; cards only where the whole card is the interactive unit (project links).
3. **One accent story** — primary teal for links, focus, and key highlights; badges stay neutral/subtle.
4. **Accessible by default** — semantic HTML in layouts, visible focus states (Nuxt UI defaults), WCAG AA contrast for text on surfaces.
5. **Content is the product** — blog and résumé remain scannable; home hero states name + role + proof without noise.
